4. The Console 控制台

控制台是启动WinAutomation时看到的第一个窗口。 从这里，您将访问该软件的所有其他部分。 在这里，您可以执行除构建或编辑流程以外的大多数任务。

窗口的7个部分是：

  1. 处理标签
  2. 触发器选项卡
  3. 计划程序选项卡
  4. 日志标签
  5. 选项标签
  6. 文件夹窗格：在此窗格中，您可以查看您的进程文件夹结构。
  7. 进程窗格：在此窗格中，您可以查看保存在特定文件夹中的所有进程。 您可以在“进程”的“文件夹”窗格中单击一个文件夹，并且其所有进程都将在“进程”的窗格中列出。

4.1 Processes 流程

在WinAutomation控制台的“进程”选项卡中，您将找到管理进程的所有选项，下面的主题对此进行了详细说明。

4.1.1 Process Tab 流程标签

“进程”选项卡具有四个菜单组和两个窗格，如下所示：

  1. Creat - 建立

    在创建组中，您可以创建一个新流程，创建一个新文件夹以将您的流程保存在控制台中，将一个流程导入/导出一个流程。

  2. Edit - 编辑

    在“编辑”组中，可以通过单击“流程设计器”选项在“流程设计器”中编辑一个流程，查看其属性，删除一个流程，在您当前所在的文件夹中选择“所有流程”，重命名/剪切/复制/粘贴过程并启用或禁用它。

  3. Execution - 执行

    在执行组中，您可以从控制台执行-运行-启动进程，并在完成之前随时停止它。

  4. Find - 查找

    在“查找”组中，您只需输入流程名称中包含的一些文本，即可在“流程”文件夹结构中搜索您的流程。

  5. Compiler - 编译器

    使用专业版，您可以创建可以在任何计算机上运行的独立可执行文件。我们稍后将进一步讨论可执行文件。

  6. The Processes Folders List pane - “进程文件夹列表”窗格

    在此窗格中，您可以查看已保存的Processes文件夹结构。

  7. The Processes List pane - “进程列表”窗格

    在此窗格中，您可以查看保存在特定文件夹中的所有进程。您可以在“进程”的“文件夹”窗格中单击一个文件夹，并且其所有进程都将在“进程”的窗格中列出。如果您根据需要的显示单击并拖动它们，则可以重新排列“过程列表”窗格中的列。

4.1.2 Create a new Folder 新建一个文件夹

要在控制台中创建新文件夹：

-在“进程”选项卡的“创建”菜单组中双击“新建文件夹”选项

或者

-在“文件夹列表”窗格中右键单击，然后选择“新建文件夹”。

新文件夹可以重命名。

任何新文件夹都将在“文件夹列表”窗格中当前选择的父文件夹下创建为子文件夹。 因此，无法在与“我的进程”相同的级别上创建另一个文件夹，因为该文件夹是根文件夹。

可以通过在“文件夹列表”窗格中将文件夹拖放到其他文件夹中来重新排列文件夹结构。 当然，在同一父文件夹下不可能有两个具有相同名称的文件夹。

4.1.3 Create a new Process 新建一个流程

您可以通过按Ctrl + N或单击“进程”选项卡上的“新建进程”按钮

来创建新进程。 这将打开“创建新流程”对话框，提示输入新流程的名称。

设置进程名称后，您有三个选择。

您可以打开：

  1. 流程设计器，然后通过在工作区中拖放动作来开始构建流程。


2. 您可以跳转到“宏记录器”以记录您的鼠标单击和击键，或智能记录任何桌面活动，然后返回到“流程设计器”，或者


3.最后，您可以打开Web Recorder，并开始在浏览器窗口中导航，在该窗口中将记录所有Web活动，并在完成记录并返回到Process Designer时将其转换为动作。

请注意，您始终可以进入“流程设计器”并从“流程设计器”窗口中访问“记录工具”（Macro Recorder和Web Recorder）。这些工具可在Process Designer的工具栏中和“工具”菜单下使用。

要在以后重新编辑流程，请双击该流程，或选择它并单击“流程”选项卡上的“流程设计器”按钮或右键单击该流程并选择“编辑流程”。

4.1.4 Move Processes/Folders 移动流程和文件夹

“文件夹列表”和“进程列表”窗格均使用“拖放”，因此移动文件夹和进程很容易。

只需在文件夹列表窗格中选择要移动的文件夹并将其放在目标文件夹中即可。 如果要移动流程，请在“流程列表”窗格中单击该流程并将其拖到适当的文件夹中。

您始终可以剪切（或复制）该过程并将其粘贴到所需的文件夹中。

4.1.5 Import/Export a Process 导入与导出流程

WinAutomation将所有进程存储在数据库中。 因此，没有明显的要复制或移动的过程文件。 要将一个进程复制到另一台装有WinAutomation的计算机上，请导出该进程-这将创建一个扩展名为.waj的文件。

可以使用WinAutomation将该文件移动到另一台计算机，然后导入到其数据库中。 现在，新计算机还将具有该进程的副本。 这是通过“流程”选项卡中的“导入和导出流程”选项完成的。

导入过程：

要在WinAutomation控制台中导入进程，请启动WinAutomation控制台，然后从“进程”选项卡的“创建”子菜单组中选择“导入进程”选项。这样做后，将出现一个对话框，提示您选择要导入的进程文件（* .waj）。

选择文件并按下“打开”按钮后，对话框窗口将关闭，并且在WinAutomation控制台中，在“过程”面板中，您将看到选择导入的过程。

汇出程序：

要导出流程，请找到要导出的流程，然后从“流程列表”窗格中，然后在“创建”菜单组中选择“导出流程”选项。这样做后，将出现一个对话框，提示您选择要将流程导出到的文件夹。

一旦选择了文件夹并在对话框中按下“确定”按钮，对话框窗口将关闭，并且在您选择的文件夹内将创建一个新文件，其名称以Process的名称命名，并带有* .waj扩展名。

如果进程包含带有密码的操作（例如，连接到Ancora），则将要求输入密码以加密上述密码并增强导出文件的安全性。首次执行导入的流程以及每次对其进行编辑时，都需要此密码。

4.1.6 Run/Stop a Process 运行与停止流程

运行/停止流程
您可以通过单击绿色箭头图标“开始”来手动启动流程，或者右键单击流程并选择运行流程。您要启动的进程必须突出显示。如果没有突出显示“进程”，则“开始”“”和“停止”“”选项不可用。

如果启动已运行的进程，则默认情况下将启动该进程的另一个实例。如果您不希望某个流程能够同时运行多个实例，则可以通过该流程的“流程属性”对话框的“并发策略”选项卡来指定。您可以通过单击“进程”选项卡中的“停止”图标，或右键单击“进程”并选择“停止进程”来停止正在运行的进程。

您也可以通过单击“通知弹出窗口”上的“停止流程”链接来停止正在运行的流程的一个实例，尽管这只会关闭该流程的该实例。

您也可以通过单击“通知弹出窗口”上的“停止流程”链接来停止正在运行的流程的一个实例，尽管这只会关闭该流程的该实例。

最后，您可以通过输入Ctrl + Alt + T同时关闭所有正在运行的进程。此HotKey是预定义的触发器，如果​​WinAutomation正在控制您的鼠标，因此您不能单击任何东西（即“紧急按钮”），则特别有用。您可以在“选项>设置>常规”选项卡中更改此热键组合。
